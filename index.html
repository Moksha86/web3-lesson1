üîë Symmetric Cryptography Fundamentals

‚ÄúSymmetric cryptography is like a shared secret handshake‚Äîboth parties know the same secret code.‚Äù ü§ù

Definition & Principles

Symmetric encryption uses a single secret key for both encryption and decryption. It is renowned for its speed and efficiency, making it ideal for bulk data encryption. Key concepts:

  • Secret Key: A random string of bits agreed upon by sender and receiver.
  • Encryption Algorithm: Transforms plaintext into ciphertext using the key.
  • Decryption Algorithm: Reverses the transformation using the same key.

Code Example: AES-256-CBC in Node.js

// Node.js: AES-256-CBC Symmetric Encryption Example
const crypto = require('crypto');
const algorithm = 'aes-256-cbc';
// 32 bytes key and 16 bytes IV
const key = crypto.randomBytes(32);
const iv = crypto.randomBytes(16);

// Encrypt function
function encrypt(text) {
  const cipher = crypto.createCipheriv(algorithm, key, iv);
  let encrypted = cipher.update(text, 'utf8', 'hex');
  encrypted += cipher.final('hex');
  return encrypted;
}

// Decrypt function
function decrypt(encryptedHex) {
  const decipher = crypto.createDecipheriv(algorithm, key, iv);
  let decrypted = decipher.update(encryptedHex, 'hex', 'utf8');
  decrypted += decipher.final('utf8');
  return decrypted;
}

// Demo
const message = "Hello, Symmetric World!";
const encryptedMsg = encrypt(message);
const decryptedMsg = decrypt(encryptedMsg);

console.log("Encrypted:", encryptedMsg);
console.log("Decrypted:", decryptedMsg);

Security Considerations

Symmetric encryption‚Äôs main challenge is secure key distribution. If the secret key is intercepted, the attacker can decrypt all messages. Common mitigation strategies include:

  • Key exchange protocols (e.g., Diffie-Hellman).
  • Secure channels (TLS) to transport keys.
  • Hardware security modules (HSMs) for key storage.
"Never reuse IVs with the same secret key. It can lead to catastrophic vulnerabilities."

üóùÔ∏è Asymmetric Cryptography Fundamentals

‚ÄúAsymmetric cryptography is like sending a locked mailbox‚Äîanyone can drop a message inside, but only the mailbox owner can open it.‚Äù üì¨

Definition & Principles

Asymmetric encryption uses a public/private key pair rather than a single shared secret. Key concepts:

  • Public Key: Shared openly for encryption.
  • Private Key: Kept secret for decryption.
  • Key Pair Generation: Mathematically linked but computationally infeasible to derive private from public.

Code Example: RSA in Node.js

// Node.js: RSA Asymmetric Encryption Example
const crypto = require('crypto');

// Generate key pair
const { publicKey, privateKey } = crypto.generateKeyPairSync('rsa', {
  modulusLength: 2048,
  publicKeyEncoding: { type: 'spki', format: 'pem' },
  privateKeyEncoding: { type: 'pkcs8', format: 'pem' }
});

const message = "Hello, Asymmetric World!";

// Encrypt with public key
const encryptedData = crypto.publicEncrypt(publicKey, Buffer.from(message));
// Decrypt with private key
const decryptedData = crypto.privateDecrypt(privateKey, encryptedData);

console.log("Encrypted:", encryptedData.toString('hex'));
console.log("Decrypted:", decryptedData.toString());

Security Considerations

Asymmetric cryptography protects against key distribution issues but introduces computation overhead. Mitigation tips:

  • Use recommended curve parameters (e.g., secp256r1, secp256k1).
  • Rotate keys regularly to limit exposure.
  • Implement proper padding schemes (OAEP for RSA).
"Always verify public key fingerprints over a trusted channel to prevent MITM attacks."
