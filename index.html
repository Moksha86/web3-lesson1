1. What Is Symmetric Cryptography? üîë

"Symmetric cryptography is like a shared secret handshake ‚Äî both parties know the same secret code." ü§ù

Definition: A single, shared secret key is used to both encrypt and decrypt information.

How It Works:

  • Two parties (e.g., Alice and Bob) agree on a secret key.
  • Alice uses the key to transform readable data (plaintext) into an encrypted form (ciphertext).
  • Bob uses the same key to decrypt the ciphertext back into the original message.
// Simplified example of symmetric encryption with AES in JavaScript
const crypto = require('crypto');
const algorithm = 'aes-256-cbc';
const key = crypto.randomBytes(32);
const iv = crypto.randomBytes(16);

function encrypt(text) {
  const cipher = crypto.createCipheriv(algorithm, key, iv);
  let encrypted = cipher.update(text, 'utf8', 'hex');
  encrypted += cipher.final('hex');
  return encrypted;
}

function decrypt(encrypted) {
  const decipher = crypto.createDecipheriv(algorithm, key, iv);
  let decrypted = decipher.update(encrypted, 'hex', 'utf8');
  decrypted += decipher.final('utf8');
  return decrypted;
}

const message = "Hello, Symmetric Crypto!";
const encryptedMsg = encrypt(message);
const decryptedMsg = decrypt(encryptedMsg);

console.log({ encryptedMsg, decryptedMsg });

Popular Algorithms:

  • AES (Advanced Encryption Standard)
  • 3DES (Triple Data Encryption Standard)
  • ChaCha20

2. What Is Asymmetric Cryptography? üóùÔ∏è

"Asymmetric cryptography is like sending a locked mailbox ‚Äî anyone can drop mail inside, but only the mailbox owner has the key." üì¨

Definition: Uses a pair of mathematically related keys: a public key (shared openly) and a private key (kept secret).

How It Works:

  • Bob generates a public/private key pair.
  • Bob shares his public key with Alice.
  • Alice encrypts the message using Bob‚Äôs public key.
  • Only Bob can decrypt the message using his private key.
// Example of asymmetric encryption (RSA) using Node.js
const crypto = require('crypto');
const { publicKey, privateKey } = crypto.generateKeyPairSync('rsa', {
  modulusLength: 2048,
});

const message = "Hello, Asymmetric Crypto!";
const encryptedData = crypto.publicEncrypt(publicKey, Buffer.from(message));
const decryptedData = crypto.privateDecrypt(privateKey, encryptedData);

console.log(decryptedData.toString());

Popular Algorithms:

  • RSA (Rivest-Shamir-Adleman)
  • ECC (Elliptic Curve Cryptography)
  • Diffie-Hellman Key Exchange

3. Use Cases in Blockchain & Web3 üåü

The combination of symmetric and asymmetric cryptography allows blockchains and Web3 applications to securely transmit, store, and verify data.

Common scenarios:

  • Data-at-Rest Encryption: Symmetric keys are used to encrypt data stored on servers and databases, ensuring confidentiality of backups.
  • Data-in-Transit Encryption: Asymmetric cryptography secures connections (TLS/SSL) while symmetric keys handle bulk data encryption inside these tunnels.
  • Wallets & Transactions: Private/public key pairs sign and authorize transactions on blockchains.
  • Messaging & Privacy: Encrypted messaging protocols combine asymmetric keys for key exchange and symmetric keys for message encryption.
// Sample Web3 wallet signature verification pseudocode
function verifySignature(message, signature, publicKey) {
  // Uses asymmetric cryptography to verify
  return crypto.verify('SHA256', message, publicKey, signature);
}

Additional Code Examples (Node.js)

Explore detailed cryptographic function examples, focusing on encryption, decryption, and signature generation/verification.

// Generate key pairs for asymmetric crypto
const { generateKeyPairSync, publicEncrypt, privateDecrypt, sign, verify } = require('crypto');

const { publicKey, privateKey } = generateKeyPairSync('rsa', {
  modulusLength: 2048,
});

const data = "Sample data for encryption";

// Encrypt using public key
const encrypted = publicEncrypt(publicKey, Buffer.from(data));

// Decrypt using private key
const decrypted = privateDecrypt(privateKey, encrypted);

console.log("Decrypted data:", decrypted.toString());

// Signing data
const signature = sign("sha256", Buffer.from(data), privateKey);

// Verifying signature
const isVerified = verify("sha256", Buffer.from(data), publicKey, signature);

console.log("Signature valid?", isVerified);
