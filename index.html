🔑 What Is Symmetric Cryptography?

Symmetric cryptography is like a shared secret handshake — both sides know the code. 🤝

Definition: A single, shared key is used for both encryption and decryption.

🔄 How It Works

  • Alice and Bob agree on a secret key 🔐
  • Alice encrypts plaintext → ciphertext using the key
  • Bob uses the same key to decrypt ciphertext → original message

🗝️ What Is Asymmetric Cryptography?

Asymmetric cryptography is like sending a locked mailbox — anyone can drop in mail, but only the mailbox owner holds the key. 📬

Definition: Two distinct keys: public key (shared) and private key (secret).

🔄 How It Works

  • Bob publishes his public key 📢
  • Alice encrypts message using Bob’s public key → ciphertext
  • Only Bob’s private key can decrypt ciphertext → plaintext

✨ Summary & Next Steps

Symmetric = one key, fast, but key sharing is tricky.
Asymmetric = two keys, better distribution and digital signatures, but slower.
Both complement in Web3: asymmetric helps exchange symmetric key for fast encryption.
